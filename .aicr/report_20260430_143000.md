### DEFECT-2 — pop() 后 push() 导致编辑跳转失效

**类型**：生命周期错误 / 功能失效

`_onEdit` 方法执行顺序：
1. `Navigator.of(context).pop()` — 同步卸载当前 Widget

### DEFECT-10 — _masterFuture 失败后不重置，导致永久加密失效
**文件**：`lib/data/crypto_service.dart:33`
**风险**：🔴 高风险（安全关键）
**类型**：缓存失败 Future / 永久不可用

```dart
_masterFuture ??= loadOrCreateMaster()
```
`_masterFuture` 一旦被赋值（即使 Future 后来以异常完成），`??=` 在后续调用时跳过重赋值，每次 `await _masterFuture` 都重新抛出相同异常。

Keystore 临时故障（如首次启动时的短暂锁定）后，只要 App 不重启，所有加密/解密操作永久失败，用户无法读取任何已加密数据。`clearCache()` 可重置但仅在登出/测试时调用。

**修复建议**：捕获失败后将 `_masterFuture` 重置为 null（让下次重试）：
```dart
_masterFuture ??= loadOrCreateMaster().catchError((e) {
  _masterFuture = null;
  throw e;
});
```
