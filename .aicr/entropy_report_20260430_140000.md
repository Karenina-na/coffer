### 空 catch 块（异常被隐式吞掉）

#### 3. `DriftExchangeRateRepository.querySeriesForPair()` 吞 DB 异常

| 文件 | 行号 | 代码片段 |
|------|------|----------|
| `data/repositories/drift_exchange_rate_repository.dart` | L43 | `} catch (_) { return const []; }` |

DB 异常被静默吞掉，调用方收到空列表——与"真正没有数据"完全无法区分。
图表渲染、汇率预警等下游模块会把 DB 故障误判为"无历史记录"，掩盖 schema 迁移失败等存储错误。

**建议**：改为 `return Err(StorageError(...))` 或至少 `debugPrint`，让调用方能区分"空数据"与"查询失败"。

#### 4. `DriftExchangeRateRepository.queryForDate()` 吞 DB 异常

| 文件 | 行号 | 代码片段 |
|------|------|----------|
| `data/repositories/drift_exchange_rate_repository.dart` | L60 | `} catch (_) { return null; }` |

`null` 既表示"这一天无汇率记录"，也表示"查询时 DB 出错"。
`RefreshAssetPriceUseCase.refreshHistory` 在查不到历史汇率时会降级使用 spot rate——DB 故障也会触发相同降级，导致历史市值用错误汇率写入而无法察觉。

**建议**：同上，改为可区分空值与错误的返回类型（如 `Result<ExchangeRate?, AppError>`）。

#### 5. `BiometricAuth.authenticate()` 吞平台锁定异常

| 文件 | 行号 | 代码片段 |
|------|------|----------|
| `core/auth/biometric_auth.dart` | L51 | `} catch (_) { return false; }` |

`local_auth` 抛出的 `PlatformException` 包含多种语义：`NotAvailable`（无硬件）、`LockedOut`（临时锁定）、`PermanentlyLockedOut`（永久锁定，需 PIN 重置）。
当前实现将所有异常与"用户主动取消"混为一谈，返回相同的 `false`。
在生物识别被永久锁定时，用户会直接跳转到 PIN 界面，而不会看到"生物识别已被锁定，请用 PIN 解锁"的提示，影响用户体验和安全感知。

**建议**：捕获 `PlatformException` 后按 `code` 区分返回值，或返回枚举（`BiometricResult.success/cancelled/locked`）。
