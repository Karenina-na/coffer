## 缺陷详情

### [DEFECT-1] 🔴 latestRes null dereference 导致同步页面崩溃
**文件**：`lib/features/asset/presentation/asset_detail_page.dart:438`
**函数**：`_syncRange`
**类型**：Null Pointer Dereference

**根因**：
```dart
// latestRes 可能为 null（当 refreshLatest() 抛出异常时）
latestRes.when(ok: (_) {}, err: (e) => errs.add('最新：${e.message}'));
//      ↑ Null check operator used on a null value
```

`histRes` 在同一函数第435行有 `if (histRes != null)` 保护，但 `latestRes` 没有。try-finally 结构中 finally 不会终止后续执行，控制流直接到达第438行。

**修复**：
```dart
latestRes?.when(ok: (_) {}, err: (e) => errs.add('最新：${e.message}'));
```

### [DEFECT-2] 🔴 id.substring(0,6) 在短 id 上越界崩溃

### [DEFECT-5] 🔴 PIN 失败计数竞态可绕过 Lockout
**文件**：`lib/core/auth/pin_store.dart:143`
**函数**：`verifyPin`
**类型**：TOCTOU 竞态

**根因**：
```dart
final fails = (int.tryParse(await _storage.read(_kFailCount) ?? '') ?? 0) + 1;
// ↑ await 点：两个并发调用均读到 failCount=4，各自写回 5
await _kdf.derive(...); // 数百毫秒 await，竞态窗口
await _storage.write(_kFailCount, fails.toString());
```

**修复**：用 `Completer` 或 `Mutex` 将整个 `verifyPin` 序列化，确保 read-increment-write 的原子性。
