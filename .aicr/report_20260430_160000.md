## 缺陷详情

### DEFECT-1 🔴 `pair_detail_page.dart` — changePct! NPE

**类型**：Null check operator used on null value
**函数**：`_PairHero.build`
**风险代码**：`'${changePct!.toStringAsFixed(2)}%'`

**问题**：`changePct` 仅在 `prev != 0` 时赋值，但外层保护仅检查 `changeAbs != null`。当 `series.length >= 2` 且 `prev == 0` 时，`changeAbs != null`（分支成立）但 `changePct == null`，`changePct!` 触发 NPE。

**修复**：将条件改为 `if (changeAbs != null && changePct != null)` 或将 `changePct!` 改为 `changePct?.toStringAsFixed(2) ?? '—'`。

### DEFECT-2 🔴 `card_detail_sheet.dart` — pop 后用失效 context

**类型**：Use-after-pop BuildContext
**函数**：`_confirmDelete`
**风险代码**：`Navigator.of(context).pop(); ScaffoldMessenger.of(context).showSnackBar(...)`

**问题**：`pop()` 将底部弹层 deactivated，之后立即调用 `ScaffoldMessenger.of(context)` 会查找不到 `ScaffoldMessenger` 祖先，抛出 `FlutterError`。

**问题**：`cast()` 是惰性的，调用时不检查类型，迭代消费时才抛 `CastError`。此时可能已部分写入数据库，且错误以 `StorageError` 而非 `ValidationError` 呈现，语义不清晰。
