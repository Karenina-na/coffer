### DEFECT-1 🟡 中风险 — 备份恢复 KDF 参数无范围校验

**文件**: `lib/domain/usecases/backup_restore.dart:148`
**函数**: `call (ImportBackupUseCase)`
**类型**: 不可信输入强制类型转换 / 拒绝服务

**风险代码**:
```dart
memoryKib: params['m'] as int,
iterations: params['t'] as int,
```

**根因**: 导入备份包时，`kdf.params` 中的 `m`/`t`/`p`/`l` 字段直接使用 `as int` 强制转换，没有任何上限约束。`PasswordKdf` 构造函数中仅有 `assert(memoryKib > 0)` 且 `assert` 在 release 模式下被忽略。攻击者可构造 `memoryKib: 2^31-1`（约 2TB）的恶意备份文件，导致 Argon2id 初始化申请海量内存，在移动端触发 OOM 崩溃。

**攻击路径**: 构造恶意备份文件 → 受害者点击「导入备份」→ jsonDecode 解析 → params 直接强转 int → PasswordKdf(memoryKib: 极大值) → Argon2id OOM 崩溃

**现有保护**: v2 AAD 绑定防止 KDF 参数被篡改后成功解密（但不防崩溃）；外层 try/catch 捕获 TypeError（类型不符时），但不阻止 OOM。

**修复建议**:
```dart
// 添加范围校验，在使用前验证 KDF 参数合理性
const _kMaxMemoryKib = 512 * 1024; // 512 MiB
const _kMaxIterations = 20;
const _kMaxParallelism = 4;
const _kMaxHashLength = 64;

final m = params['m'];
final t = params['t'];
final p = params['p'];
final l = params['l'];
if (m is! int || m <= 0 || m > _kMaxMemoryKib ||
    t is! int || t <= 0 || t > _kMaxIterations ||
    p is! int || p <= 0 || p > _kMaxParallelism ||
    l is! int || l < 16 || l > _kMaxHashLength) {
  return const Err(ValidationError('invalid kdf params'));
}
final kdf = PasswordKdf(memoryKib: m, iterations: t, parallelism: p, hashLength: l);
```
